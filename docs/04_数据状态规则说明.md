# 04 数据状态规则说明

## 通用启停与软删除

多数核心业务表包含以下字段：

- `enabled`：是否启用。检索和问答通常只使用 `enabled=true` 的知识。
- `deleted`：是否软删除。列表、详情和检索默认排除 `deleted=true`。
- `created_at`、`updated_at`：创建和更新时间。
- `version`：部分表预留版本字段，当前未实现乐观锁。

删除规则：

- 结构化知识删除时执行软删除：`deleted=true`、`enabled=false`、`updated_at=now()`。
- 文档切片重新解析时会物理删除该文档旧切片，再写入新切片。
- 任务日志通过外键 `ON DELETE CASCADE` 跟随任务删除，但当前业务没有任务删除接口。

## 项目状态

`ai_project.status` 当前前端默认使用：

- `active`：正常项目。
- `archived`：归档项目。

项目是否可检索还受 `enabled` 和 `deleted` 影响。

## 文档解析状态

`ai_document.parse_status` 当前使用以下状态：

| 状态 | 含义 | 进入方式 |
| --- | --- | --- |
| `uploaded` | 已上传，未解析 | 上传文档后默认状态 |
| `parsing` | 解析中 | 执行解析任务时设置 |
| `ready` | 解析成功，可用于切片查看和检索 | 文本提取、切片、入库完成后设置 |
| `failed` | 解析失败 | Tika 提取失败、无有效文本或其他异常 |

解析失败时：

- 原始文件保留。
- `parse_error` 记录失败原因。
- `last_parsed_at` 不更新为成功时间。
- 可通过任务重试重新解析。

## 文档索引状态

`ai_document.index_status` 和 `ai_document_chunk.index_status` 当前使用：

| 状态 | 含义 |
| --- | --- |
| `pending` | 待索引或刚重建切片后待处理 |
| `indexed` | 已完成 embedding/向量索引写入 |
| `partial_failed` | 部分切片索引成功，部分失败 |
| `failed` | 索引失败 |

索引规则：

- 文档解析成功后会尝试索引。
- 单个切片索引失败会记录 `index_error`。
- 文档级索引状态根据切片成功/失败数量汇总。
- 索引失败不影响关键词检索，系统会降级使用关键词。
- 批量重建索引只处理 `deleted=false`、`enabled=true`、`parse_status=ready` 的文档。

## 任务状态

`ai_task.status` 当前使用：

| 状态 | 含义 |
| --- | --- |
| `pending` | 已创建，等待执行 |
| `running` | 执行中 |
| `success` | 执行成功 |
| `failed` | 执行失败 |
| `cancelled` | 已取消 |

任务规则：

- 新任务默认 `progress=0`。
- 任务开始后设置为 `running`，进度至少为 1。
- 执行中进度最大限制为 99，成功后统一设为 100。
- 当前只支持取消 `pending` 状态任务。
- 当前只支持重试 `failed` 状态任务。
- 任务日志按创建时间正序展示。

## 反馈状态

`ai_feedback.review_status` 当前使用：

| 状态 | 含义 |
| --- | --- |
| `pending` | 待复核 |
| `accepted` | 已采纳 |
| `rejected` | 已驳回 |
| `converted` | 已转入知识库草稿 |

反馈转知识库规则：

- 必须存在合法 `target_knowledge_type`。
- 转入知识库后生成的知识默认 `enabled=false`。
- 转入成功后反馈状态改为 `converted`，`convert_to_knowledge=true`。

## 能力支持状态

`ai_capability.support_level` 是自由文本枚举式字段，示例数据使用：

- `supported`：已支持。
- `config_supported`：通过配置可支持。
- `custom_required`：需要定制或二次开发。

需求分析会优先使用能力清单判断是否可支持。

## 接口状态

`ai_api.status` 当前默认 `active`。接口是否参与检索还受 `enabled` 和 `deleted` 控制。

## 评测状态

`ai_eval_case.enabled` 控制评测用例是否启用。

`ai_eval_run.status` 当前按运行过程使用：

- `pending`：待运行。
- `running`：运行中。
- `success`：运行完成。
- `failed`：运行失败。

`ai_eval_result` 同时保存：

- `auto_score`：自动评分。
- `auto_passed`：自动是否通过。
- `manual_passed`：人工复核是否通过。
- `passed`、`score`：人工或兼容字段。

## 模型和 Prompt 状态

模型配置和 Prompt 模板均支持：

- `enabled`：是否启用。
- `default_config` 或 `default_template`：是否默认。
- `deleted`：软删除。

设置默认配置时，同类型旧默认会被清空，并自动启用当前配置。
