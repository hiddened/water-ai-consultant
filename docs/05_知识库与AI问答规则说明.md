# 05 知识库与 AI 问答规则说明

## 知识来源

系统当前检索以下知识来源：

| 来源类型 | 数据表 | 用途 |
| --- | --- | --- |
| `DOCUMENT_CHUNK` | `ai_document_chunk` | 文档原文切片 |
| `DOCUMENT` | `ai_document` | 文档名称、类型、路径和状态 |
| `PAGE` | `ai_page` | 页面操作、入口、业务规则 |
| `CAPABILITY` | `ai_capability` | 系统能力、支持等级、限制 |
| `API` | `ai_api` | 接口路径、参数、响应、鉴权 |
| `DB_TABLE` | `ai_db_table` | 表字段、表关系、数据含义 |
| `REQUIREMENT_CASE` | `ai_requirement_case` | 历史需求、方案、等级、风险 |

检索只使用当前项目下、未删除且启用的数据。文档切片还要求文档处于可用状态。

## 文档入库流程

```text
上传文件
  |
  v
保存原文件到本地 storage 或 MinIO
  |
  v
写入 ai_document，parse_status=uploaded
  |
  v
创建解析任务
  |
  v
Apache Tika 提取文本
  |
  v
DocumentChunker 切片
  |
  v
写入 ai_document_chunk
  |
  v
尝试 embedding/pgvector 索引
  |
  v
解析成功 ready；索引失败则保留切片并降级关键词检索
```

## 检索策略

检索入口为 `KnowledgeRetrievalService.retrieve(projectId, query, mode, limit)`。

主要规则：

- 先通过 `KeywordExtractor` 提取关键词。
- 关键词为空时直接返回空依据。
- 文档切片优先调用 `VectorStore.search`。
- 向量层异常时自动降级到 `DocumentChunkSearchService.keywordSearch`。
- 结构化知识按标题、模块、定位符和正文进行关键词打分。
- 最终按问答模式优先级和分数排序。

## 模式优先级

不同模式会提升不同知识类型：

- `doc_qa`：文档切片、文档、页面、能力、历史案例、接口、数据表。
- `page_help`：页面、文档切片、能力、接口、文档、历史案例、数据表。
- `business_qa`：文档切片、能力、页面、数据表、接口、历史案例、文档。
- `requirement_check`：能力、历史需求案例、页面、接口、数据表、文档切片、文档。

## 问答生成规则

问答流程：

```text
用户问题
  |
  v
知识检索
  |
  v
Prompt 模板渲染
  |
  v
Mock LLM / DeepSeek / OpenAI 兼容模型
  |
  v
结构化回答 + references
  |
  v
保存会话、消息、引用、模型信息、Prompt 信息
```

无依据时不调用真实模型，直接返回固定拒答语：

```text
当前资料不足，无法确认。建议补充相关项目文档、能力清单或历史需求案例。
```

## Prompt 模板规则

Prompt 模板支持：

- `template_type`：`chat`、`requirement_check`、`page_help`、`business_qa`、`doc_qa`。
- `mode`：可指定具体问答模式，也可为空作为通用模板。
- `output_format`：`markdown`、`json`、`structured_json`。
- `default_template`：同一类型和模式下只应有一个默认模板。

模板变量包括：

- `question`
- `project_name`
- `mode`
- `context`
- `references`
- `related_pages`
- `related_capabilities`
- `related_apis`
- `related_tables`
- `requirement_desc`
- `module_name`
- `current_time`

没有配置默认模板时，系统使用内置模板。

## 引用溯源规则

每次问答会返回 `references`，并写入 `ai_answer_reference`。引用包含：

- 来源类型。
- 来源 ID。
- 来源标题。
- 来源定位符。
- 引用片段。
- 检索分数。

前端可通过引用详情查看原文或结构化知识内容。评测也会使用引用来源判断是否命中预期依据。

## 置信度规则

当前置信度为简化规则：

- 有检索依据的普通问答：约 `0.76`。
- 有检索依据的需求分析：约 `0.74`。
- 无依据拒答：约 `0.10`。

该字段用于前端展示和后续评测，不代表严格统计概率。

## 模型规则

模型来源优先级：

1. 数据库中启用且默认的 `chat` 模型配置。
2. 环境变量中的默认模型提供方。
3. Mock 模型兜底。

Embedding 来源优先级：

1. 数据库中启用且默认的 `embedding` 模型配置。
2. 环境变量或 MockEmbeddingClient。

真实模型调用异常会返回业务错误；向量检索异常会降级关键词检索。

## 检索调试规则

`POST /api/search/debug` 只执行检索，不调用大模型。返回内容包括：

- 查询语句。
- 检索策略。
- embedding provider。
- 是否启用向量。
- 是否使用关键词兜底。
- 各类命中来源和切片详情。

该接口用于解释“为什么问答命中了这些依据”。
