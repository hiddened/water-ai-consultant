# 10 部署运行规则说明

## Docker Compose 启动

首次启动：

```bash
cp .env.example .env
docker compose up -d --build
```

默认访问地址：

- 前端：`http://127.0.0.1:3000`
- 后端：`http://127.0.0.1:8080`
- Swagger：`http://127.0.0.1:8080/swagger-ui/index.html`
- MinIO 控制台：`http://127.0.0.1:9001`

健康检查：

```bash
curl http://127.0.0.1:8080/api/health
```

停止：

```bash
docker compose down
```

清空数据库和 MinIO/PostgreSQL volume 后重建：

```bash
docker compose down -v
docker compose up -d --build
```

## Docker 服务

`docker-compose.yml` 包含：

- `postgres`：PostgreSQL 16。
- `minio`：对象存储，可选用于文档原文件。
- `water-ai-consultant-server`：Spring Boot 后端。
- `water-ai-consultant-web`：Nginx + Vue 前端。

启动依赖：

- 后端等待 PostgreSQL 健康检查通过。
- 前端等待后端健康检查通过。
- MinIO 随 Compose 启动，默认总是启动。

## 环境变量

主要变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `POSTGRES_DB` | `water_ai_consultant` | 数据库名 |
| `POSTGRES_USER` | `postgres` | 数据库用户 |
| `POSTGRES_PASSWORD` | `postgres` | 数据库密码 |
| `POSTGRES_PORT` | `5432` | 数据库宿主机端口 |
| `BACKEND_PORT` | `8080` | 后端端口 |
| `FRONTEND_PORT` | `3000` | 前端端口 |
| `LLM_PROVIDER` | `mock` | 默认模型提供方 |
| `DEEPSEEK_API_KEY` | 空 | DeepSeek API Key |
| `DEEPSEEK_BASE_URL` | `https://api.deepseek.com` | DeepSeek Base URL |
| `DEEPSEEK_MODEL` | `deepseek-v4-flash` | DeepSeek 模型名 |
| `STORAGE_TYPE` | `local` | 文件存储类型 |
| `STORAGE_PATH` | `/app/storage` | 容器内本地存储路径 |
| `MINIO_ENDPOINT` | `http://minio:9000` | 后端访问 MinIO 地址 |
| `MINIO_BUCKET` | `water-ai-consultant` | MinIO bucket |
| `EMBEDDING_PROVIDER` | `mock` | Embedding 提供方 |
| `VECTOR_PROVIDER` | `pgvector` | 向量存储提供方 |

不要提交真实 `.env`，仓库已忽略该文件。

## Mock 模式

默认使用：

```env
LLM_PROVIDER=mock
EMBEDDING_PROVIDER=mock
```

适用场景：

- 本地启动。
- 页面演示。
- 接口联调。
- 评测流程验证。

Mock 模式不代表真实问答效果。

## DeepSeek 模式

配置：

```env
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的 Key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
```

也可以通过前端“模型配置”维护数据库默认模型。数据库默认模型优先级高于环境变量兜底模型。

## 文件存储

本地存储：

```env
STORAGE_TYPE=local
STORAGE_PATH=/app/storage
```

Docker 会把宿主机 `./storage` 挂载到容器内。仓库只保留 `.gitkeep`，不会提交上传文件。

MinIO 存储：

```env
STORAGE_TYPE=minio
MINIO_ENDPOINT=http://minio:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=water-ai-consultant
```

MinIO 只保存原文件，文档切片、引用、索引状态仍保存在 PostgreSQL。

## 本地开发运行

数据库：

```bash
createdb water_ai_consultant
psql -U postgres -d water_ai_consultant -f database/init.sql
```

后端：

```bash
cd backend
mvn spring-boot:run
```

如数据库连接不是默认值：

```bash
set DB_URL=jdbc:postgresql://127.0.0.1:5432/water_ai_consultant
set DB_USERNAME=postgres
set DB_PASSWORD=你的密码
mvn spring-boot:run
```

前端：

```bash
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 3000
```

## Nginx 代理规则

前端容器的 Nginx 代理：

- `/api/` -> `water-ai-consultant-server:8080/api/`
- `/swagger-ui/` -> `water-ai-consultant-server:8080/swagger-ui/`
- `/v3/api-docs` -> `water-ai-consultant-server:8080/v3/api-docs`
- 其他路径回退到 Vue 单页应用入口。

## 数据初始化规则

Docker PostgreSQL 只在数据目录为空时执行 `/docker-entrypoint-initdb.d`。如果修改了初始化 SQL 或 seed 数据，需要清空 volume：

```bash
docker compose down -v
docker compose up -d --build
```

## 生产化注意事项

当前项目默认配置偏演示环境，生产部署前至少需要补充：

- 登录鉴权和接口权限控制。
- 用户-项目数据权限。
- API Key 加密存储。
- HTTPS 和反向代理安全头。
- 数据库、MinIO、后端服务仅内网暴露。
- 结构化日志、监控、告警和备份策略。
- 上传文件大小、类型白名单和安全扫描。
- 真实模型调用限流、超时和重试策略。
- 数据库迁移工具，如 Flyway 或 Liquibase。
