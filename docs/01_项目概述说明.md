# 01 项目概述说明

## 项目定位

`water-ai-consultant` 是面向水务行业内部团队的 AI 项目智能顾问平台，主要服务售前、产品、实施、开发、运维等内部角色。系统不是面向市民的客服系统，也不是单一文档问答工具，而是围绕项目资料、系统能力、页面操作、接口、数据库说明、历史需求案例和评测结果形成内部知识辅助平台。

## 核心目标

- 统一沉淀水务项目知识，减少项目资料分散导致的重复沟通。
- 支持基于项目知识库的智能问答、页面操作问答、业务规则问答和接口问答。
- 支持需求可行性分析，按项目已有能力和历史案例输出可行性等级、风险点和建议方案。
- 支持文档上传、解析、切片、索引和引用溯源。
- 支持问答反馈、反馈复核和反馈转知识库草稿。
- 支持评测用例、批量评测、自动评分和回归看板。
- 支持模型配置、Prompt 模板配置、本地文件存储和 MinIO 对象存储。

## 技术栈

- 后端：Spring Boot 3.3、Java 21、Spring JDBC、Spring Validation、Springdoc OpenAPI。
- 前端：Vue 3、Vite、TypeScript、Nginx。
- 数据库：PostgreSQL，支持在安装 pgvector 时启用向量列和向量索引。
- 文档解析：Apache Tika。
- 文件存储：本地 `storage/` 或 MinIO。
- 模型调用：默认 Mock LLM，可配置 DeepSeek 或 OpenAI 兼容模型。
- 部署方式：Docker Compose 一键启动。

## 系统模块

- Dashboard：查看健康状态、文档、任务和评测概况。
- 智能问答：根据项目知识回答文档、业务、页面、接口相关问题。
- 需求可行性分析：结合能力清单和历史需求案例输出等级判断。
- 检索调试：只执行检索，不调用大模型，用于分析命中来源。
- 文档知识库：上传文档、异步解析、查看切片、下载原文件、重建索引。
- 结构化知识管理：项目、页面说明、能力清单、接口说明、数据表说明、历史需求案例。
- 对话记录：保存问答会话、消息、模型信息、Prompt 信息和引用来源。
- 反馈管理：提交反馈、复核反馈、转入知识库草稿。
- 模型配置：维护 chat/embedding 模型配置，支持设置默认配置和连接测试。
- Prompt 模板：维护不同问答模式的默认模板和渲染测试。
- 评测与回归：维护评测用例、运行单例/批量评测、自动评分、人工复核和失败重跑。
- 任务中心：记录文档解析、重建索引等异步任务进度和日志。

## 运行拓扑

```text
浏览器
  |
  | http://127.0.0.1:3000
  v
Nginx + Vue dist
  |
  | /api, /swagger-ui, /v3/api-docs
  v
Spring Boot API
  |-------- 本地 storage 或 MinIO
  |-------- Apache Tika
  |-------- Mock LLM / DeepSeek / OpenAI 兼容模型
  v
PostgreSQL
```

## 当前实现边界

- 当前项目没有登录、鉴权和角色权限模块，所有接口默认可访问。
- 当前默认模型为 Mock，适合演示和联调，不代表真实问答质量。
- 真实向量检索依赖 pgvector 和真实 embedding 配置；缺失时系统会降级到关键词检索。
- 文档上传后不会自动同步删除原始文件，删除知识记录采用数据库软删除。
- 多租户/多项目隔离目前主要依赖 `project_id` 查询过滤，尚未实现用户级权限隔离。
