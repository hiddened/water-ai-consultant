# AI 水务项目智能顾问平台

## 项目描述

AI 水务项目智能顾问平台是一个面向水务行业售前、产品、实施、开发和运维人员的内部智能顾问系统，用于解决定制化项目中文档分散、历史需求难复用、页面操作说明难查、客户需求可行性判断依赖人工经验的问题。系统基于 Spring Boot、Vue 3、PostgreSQL、Apache Tika 和 DeepSeek，实现项目资料管理、文档上传解析切片、RAG 智能问答、需求可行性分析、引用溯源、检索调试、问答评测、自动评分、反馈复核和反馈转知识库，并支持 Docker Compose 本地一键启动和演示。

## 个人职责

1. 负责 Spring Boot 后端整体架构设计，完成统一响应格式、统一异常处理、Swagger/OpenAPI 文档和健康检查接口。
2. 负责 PostgreSQL 数据库表设计，覆盖项目、文档、文档切片、页面说明、能力清单、接口说明、数据表说明、历史需求案例、任务、对话、引用、反馈、评测、模型配置和 Prompt 模板。
3. 负责结构化知识 CRUD 能力建设，实现项目管理、文档管理、页面说明、能力清单、接口说明、数据表说明和历史需求案例维护。
4. 负责文档上传、Apache Tika 文本提取、文档切片入库和 chunk 来源信息保存，支持问答时定位到文档和切片。
5. 负责异步任务中心设计，实现文档解析、重解析、索引任务、任务状态、进度、错误信息、日志查看、重试和取消 pending 任务。
6. 负责 Spring AI 主链路改造，使用 `ChatClient`、`PromptTemplate`、结构化输出转换器和 `EmbeddingModel` 接入 DeepSeek/OpenAI 兼容模型。
7. 负责智能问答 `/api/chat` 和需求可行性分析 `/api/requirements/check`，实现检索、Prompt 组装、模型调用、结构化返回和无依据拒答。
8. 负责引用溯源设计，将回答依据保存到 `ai_answer_reference`，前端支持引用列表和原文详情展开。
9. 负责检索调试接口 `/api/search/debug` 和前端检索调试页，用于排查文档解析、切片、检索、Prompt 和模型问题。
10. 负责问答评测和自动评分体系，实现评测用例、单条/批量运行、关键词/引用/拒答/可行性等级评分、评测运行看板和人工复核。
11. 负责反馈管理闭环，实现用户反馈、管理员复核、反馈转页面说明/能力清单/接口说明/数据表说明/历史需求案例/FAQ 草稿。
12. 负责 Docker Compose、本地启动文档、演示数据、示例文档和前端 UI 优化，支持项目本地一键启动和完整流程演示。

## 项目亮点

1. 不是普通文档问答系统，而是面向水务项目交付场景的内部智能顾问，支持问答、需求分析和历史经验复用。
2. 结合结构化知识和文档 chunk，检索来源覆盖文档、页面、能力、接口、数据库表和历史需求案例。
3. 需求可行性分析有规则护栏，能力清单和历史需求案例优先，避免模型无依据判断“已支持”。
4. 回答必须返回引用来源，无依据时固定拒答，降低大模型幻觉风险。
5. 提供检索调试、问答评测、自动评分和评测运行看板，便于持续验证 RAG 问答质量。
6. 形成“问答 -> 反馈 -> 复核 -> 转知识库”的知识沉淀闭环，并支持 Docker Compose 一键演示。
