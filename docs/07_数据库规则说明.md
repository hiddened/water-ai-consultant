# 07 数据库规则说明

## 数据库基础

项目使用 PostgreSQL。初始化脚本位于：

- 本地开发：`database/init.sql`
- Docker 初始化：`docker/postgres/init.sql`、`docker/postgres/seed-demo.sql`

初始化脚本会启用：

- `pgcrypto`：用于 `gen_random_uuid()`。
- `vector`：仅当数据库环境已安装 pgvector 时启用。

## 命名规则

- 业务表统一使用 `ai_` 前缀。
- 主键统一使用 UUID。
- 时间字段使用 `TIMESTAMPTZ`。
- JSON 字段使用 `JSONB`。
- 软删除字段为 `deleted`。
- 启用字段为 `enabled`。

## 主外键关系

- `ai_document.project_id` 关联 `ai_project.id`。
- `ai_document_chunk.project_id` 关联 `ai_project.id`。
- `ai_document_chunk.document_id` 关联 `ai_document.id`。
- 页面、能力、接口、数据表、历史需求案例均关联 `ai_project.id`。
- `ai_chat_message.session_id` 关联 `ai_chat_session.id`。
- `ai_answer_reference.message_id` 关联 `ai_chat_message.id`。
- `ai_feedback` 可关联项目、会话和消息。
- `ai_faq.source_feedback_id` 关联 `ai_feedback.id`。
- `ai_eval_result.eval_case_id` 关联 `ai_eval_case.id`。
- `ai_eval_result.run_id` 关联 `ai_eval_run.id`。
- `ai_task_log.task_id` 关联 `ai_task.id`，并设置级联删除。

## 唯一约束

- `ai_project.project_code` 唯一。
- `ai_document_chunk(document_id, chunk_no)` 唯一。

重新解析文档时，系统会物理删除旧切片再插入新切片，避免 chunk 唯一约束冲突。

## 软删除规则

多数业务表通过 `deleted` 控制软删除。列表查询、详情查询和检索默认增加：

```sql
WHERE deleted = FALSE
```

结构化资源删除时：

```sql
SET deleted = TRUE, enabled = FALSE, updated_at = now()
```

## 启用规则

`enabled=false` 的知识不应进入问答检索范围。当前检索服务对以下表显式过滤 `enabled=true`：

- `ai_document`
- `ai_page`
- `ai_capability`
- `ai_api`
- `ai_db_table`
- `ai_requirement_case`

反馈转知识库草稿默认 `enabled=false`。

## 文档存储字段

`ai_document` 同时支持本地存储和 MinIO：

- `file_path`：本地路径或逻辑路径。
- `storage_type`：`local` 或 `minio`。
- `storage_bucket`：MinIO bucket。
- `storage_object_key`：MinIO 对象 Key。
- `content_type`：文件类型。
- `file_hash`：文件哈希。
- `file_size`：文件大小。

## JSONB 字段

以下字段使用 JSONB：

- `ai_db_table.field_desc`：表字段说明。
- `ai_feedback.expected_sources`：期望引用来源。
- `ai_eval_case.expected_sources`、`expected_keywords`、`expected_source_titles`、`expected_source_types`、`score_rules`、`tags`。
- `ai_eval_result.actual_references`、`matched_keywords`、`matched_sources`、`missing_keywords`、`missing_sources`、`score_detail`。

写入 JSONB 字段时应传入合法 JSON 字符串或数组对象。

## 向量字段规则

如果 PostgreSQL 已安装 pgvector，初始化脚本会为 `ai_document_chunk` 增加：

```sql
embedding VECTOR(1536)
```

并尝试创建 ivfflat 索引。

规则：

- 默认维度为 1536。
- `EMBEDDING_DIMENSION` 必须与数据库向量列维度一致。
- 更换 embedding 模型维度时，需要同步调整数据库列和索引，并重建文档索引。
- 未安装 pgvector 时，系统仍可使用关键词检索。

## 索引规则

初始化脚本为常用查询创建索引，包括：

- 项目维度文档查询。
- 文档模块筛选。
- 文档切片项目查询。
- 页面、能力、接口、数据表、历史案例按项目/模块查询。
- 对话消息按会话和时间查询。
- 回答引用按消息查询。
- 反馈按项目、状态、时间查询。
- 评测用例、评测结果、评测运行查询。
- 任务按类型、状态、业务对象查询。
- 模型配置和 Prompt 模板默认项查询。

## 初始化数据规则

演示数据默认创建 `DEMO_WATER` 项目，并写入：

- 外部告警推送接口。
- 告警诊断 Trace 页面。
- 文档切片查看页面。
- 外部告警接入与自动建单能力。
- `biz_workorder` 表说明。
- 二供泵房告警接入并自动建单历史案例。
- 水质异常自动派单历史案例。
- 一组评测用例。

Docker 官方 PostgreSQL 镜像只在数据目录为空时执行初始化脚本。修改初始化 SQL 后，需要执行：

```bash
docker compose down -v
docker compose up -d --build
```
