# 08 异常处理规则说明

## 统一异常响应

后端通过 `GlobalExceptionHandler` 统一处理异常，返回 `ApiResponse`：

```json
{
  "code": 40000,
  "message": "错误信息",
  "data": null,
  "trace_id": "trace-id",
  "timestamp": "2026-07-02T12:00:00+08:00"
}
```

当前错误码：

| 错误码 | 含义 |
| --- | --- |
| `40000` | 请求参数、业务校验、数据约束、上游配置等错误 |
| `40400` | 数据或接口不存在 |
| `50000` | 系统内部错误、数据库不可用或上游调用失败 |

## 业务异常

业务异常通过 `BusinessException` 抛出，典型场景包括：

- `project_id` 为空。
- 问答 `mode` 不支持。
- 上传文件为空。
- 文档不存在。
- 任务不存在。
- 只有 `failed` 任务可以重试。
- 只有 `pending` 任务可以取消。
- 模型 provider 或 model_type 不支持。
- Prompt template_type 或 output_format 不支持。
- 反馈类型、复核状态、目标知识类型不支持。

业务异常当前统一返回 HTTP 400。

## 参数校验异常

以下异常会返回 `40000`：

- Bean Validation 校验失败。
- 请求参数约束失败。
- JSON 请求体格式不合法。
- UUID 格式不合法。

请求体格式不合法时返回：

```text
请求体格式不合法
```

## 数据库异常

数据库异常规则：

- 唯一约束冲突返回：`唯一字段已存在`。
- 外键、非空等完整性约束失败返回：`数据不符合数据库约束，请检查必填项和关联项目`。
- 数据库不可用或表未初始化返回：`数据库不可用或表未初始化`。

数据库访问异常会记录后端错误日志，但不会把 JDBC 细节直接暴露给前端。

## 404 异常

接口路径不存在时返回：

```text
接口不存在
```

业务数据不存在时返回对应文案，例如：

- `文档不存在`
- `任务不存在`
- `对话不存在`
- `模型配置不存在`
- `Prompt 模板不存在`
- `评测用例不存在`

## 兜底异常

未识别异常统一返回：

```text
系统繁忙，请稍后重试
```

兜底异常不向前端暴露堆栈和内部实现细节。

## 文档异常处理

文档上传异常：

- 保存失败返回 `文档保存失败：原因`。
- 文件名会先做非法字符替换，降低路径异常风险。

文档解析异常：

- 文档状态改为 `failed`。
- `parse_error` 写入异常原因。
- 任务状态改为 `failed`。
- 原始文件不删除，方便后续重试。

文档索引异常：

- 文档或切片索引状态改为 `failed` 或 `partial_failed`。
- `index_error` 写入异常原因。
- 切片不删除。
- 问答检索降级为关键词检索。

## 模型异常处理

模型配置异常：

- API Key 未配置返回 `模型 API Key 未配置` 或 `Embedding API Key 未配置`。
- Base URL 未配置返回 `模型 Base URL 未配置` 或 `Embedding Base URL 未配置`。
- Embedding 维度不匹配返回维度差异说明。

模型调用异常：

- Chat 调用失败返回 `模型调用失败：原因`。
- Embedding 调用失败返回 `Embedding 调用失败：原因`。
- 系统不会记录或返回完整 API Key。

## 问答无依据处理

问答和需求分析在无检索依据时不视为系统异常，而是业务拒答：

```text
当前资料不足，无法确认。建议补充相关项目文档、能力清单或历史需求案例。
```

此时：

- `confidence` 约为 `0.10`。
- `search_strategy=none`。
- `insufficient_answer=true`。
- 仍会保存对话记录。

## JSON 解析降级

需求分析要求模型返回 JSON。如果模型返回无法解析：

- 系统降级为 `feasibility_level=D`。
- 结论降级为 `当前资料不足，无法确认`。

该规则用于避免模型输出格式异常导致接口崩溃。

## Trace ID 规则

每个请求都会生成或传递 `trace_id`，用于：

- API 响应排查。
- 后端日志关联。
- 问答结果保存。
- 反馈和评测追踪。

日志格式中包含：

```text
[trace_id:xxx]
```
