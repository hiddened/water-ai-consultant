# 06 接口规则说明

## 通用响应格式

后端统一返回 `ApiResponse<T>`：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "trace-id",
  "timestamp": "2026-07-02T12:00:00+08:00"
}
```

字段规则：

- `code=0` 表示成功。
- 非 0 表示失败，当前主要错误码为 `40000`、`40400`、`50000`。
- `trace_id` 用于串联请求、日志、问答结果和异常排查。
- JSON 字段使用 snake_case。

## API 文档入口

- Swagger UI：`/swagger-ui/index.html`
- OpenAPI JSON：`/v3/api-docs`
- 健康检查：`GET /api/health`

## 健康检查

```http
GET /api/health
```

用于 Docker 健康检查和前端 Dashboard 状态展示。

## 结构化知识接口

以下资源支持列表、详情、新增、编辑、删除：

- `/api/projects`
- `/api/documents`
- `/api/pages`
- `/api/capabilities`
- `/api/apis`
- `/api/db-tables`
- `/api/requirement-cases`

通用规则：

- `GET /api/{resource}` 支持 `project_id`、`module_name`、`keyword` 过滤。
- `GET /api/{resource}/{id}` 查询单条。
- `POST /api/{resource}` 新增。
- `PUT /api/{resource}/{id}` 更新。
- `DELETE /api/{resource}/{id}` 软删除。

注意：文档资源同时有结构化 CRUD 和专用上传/解析接口，真实上传文件应使用 `/api/documents/upload`。

## 文档专用接口

```http
POST /api/documents/upload
```

上传文档，表单字段包括：

- `project_id`
- `module_name`
- `document_type`
- `file`

```http
POST /api/documents/{id}/parse
POST /api/documents/{id}/reindex
POST /api/documents/reindex-all
GET  /api/documents/{id}/chunks
GET  /api/documents/{id}/download
```

规则：

- 解析和重建索引通过异步任务执行，返回 `task_id`。
- 切片查看按 chunk 顺序返回。
- 下载接口返回原始文件资源。

## 智能问答接口

```http
POST /api/chat
Content-Type: application/json
```

请求示例：

```json
{
  "project_id": "uuid",
  "mode": "business_qa",
  "question": "外部告警推送接口怎么用？"
}
```

`mode` 支持：

- `doc_qa`
- `page_help`
- `business_qa`
- `requirement_check`

返回重点字段：

- `answer`
- `references`
- `related_pages`
- `related_capabilities`
- `related_apis`
- `confidence`
- `trace_id`
- `session_id`
- `message_id`
- `model_provider`
- `model_name`
- `prompt_template_id`
- `prompt_template_name`
- `search_strategy`
- `insufficient_answer`

## 需求分析接口

```http
POST /api/requirements/check
Content-Type: application/json
```

请求示例：

```json
{
  "project_id": "uuid",
  "requirement_desc": "客户想接入二供泵房告警并自动建单，能不能做？",
  "module_name": "告警中心"
}
```

返回重点字段：

- `requirement_understanding`
- `feasibility_level`
- `conclusion`
- `matched_capabilities`
- `missing_capabilities`
- `related_pages`
- `related_apis`
- `related_tables`
- `impact_modules`
- `risk_points`
- `recommended_solution`
- `workload_level`
- `references`

## 检索调试接口

```http
POST /api/search/debug
```

请求字段：

- `project_id`
- `question`
- `mode`
- `top_k`

该接口只检索，不调用模型，不保存对话。

## 对话记录接口

```http
GET    /api/chat-sessions
GET    /api/chat-sessions/{id}
GET    /api/chat-sessions/{id}/messages
DELETE /api/chat-sessions/{id}
```

列表支持项目、模式、关键词、起止时间过滤。删除为软删除会话。

## 反馈接口

```http
GET  /api/feedback
GET  /api/feedback/{id}
POST /api/feedback
PUT  /api/feedback/{id}/review
POST /api/feedback/{id}/convert
```

规则：

- 创建反馈必须包含 `project_id`、`session_id`、`message_id`。
- `feedback_type` 必须是允许值。
- 复核状态必须是允许值。
- 转知识库前必须选择合法目标知识类型。

## 模型配置接口

```http
GET    /api/model-configs
GET    /api/model-configs/{id}
POST   /api/model-configs
PUT    /api/model-configs/{id}
DELETE /api/model-configs/{id}
POST   /api/model-configs/{id}/test
POST   /api/model-configs/{id}/set-default
```

规则：

- `provider` 支持 `mock`、`deepseek`、`openai_compatible`。
- `model_type` 支持 `chat`、`embedding`。
- API Key 只返回脱敏值。

## 任务接口

```http
GET  /api/tasks
GET  /api/tasks/{id}
GET  /api/tasks/{id}/logs
POST /api/tasks/{id}/retry
POST /api/tasks/{id}/cancel
```

规则：

- 只支持取消 `pending` 任务。
- 只支持重试 `failed` 任务。
- 当前主要支持文档解析和索引重建类任务。
