# 演示指南

## 演示数据

Docker 首次启动会导入公开演示项目：

- 项目名称：智慧水务告警智能处置示范项目
- 项目编码：`DEMO_ALARM`

演示数据包括：

- 能力清单：外部告警推送、告警自动诊断、告警自动生成工单等
- 页面说明：告警诊断页、文档知识库页、检索调试页、问答评测页等
- 接口说明：`POST /api/chat`、`POST /api/requirements/check`、`POST /api/search/debug` 等
- 历史需求案例：二供泵房告警接入、水质异常告警自动派单等
- 评测用例：告警接口、参数解释、需求分析和资料不足拒答

## 推荐演示流程

1. 打开前端：http://127.0.0.1:3000
2. 进入“项目管理”，查看示例项目。
3. 进入“智能问答”，选择示例项目，提问：

   ```text
   外部告警推送接口怎么用？
   ```

4. 进入“需求可行性分析”，输入：

   ```text
   客户想接入二供泵房告警并自动建单，能不能做？
   ```

5. 进入“检索调试”，输入：

   ```text
   auto_diagnose 参数有什么作用？
   ```

6. 进入“问答评测”，点击批量运行。
7. 进入“评测运行看板”，查看通过率、平均分和低分用例。

## 文档上传演示

可以使用 `sample-docs/` 中的公开样例文档：

- `project-management-guide.md`
- `inspection-task-rules.md`
- `api-integration-notes.md`

步骤：

1. 打开“文档知识库”。
2. 选择示例项目。
3. 上传上述文档。
4. 点击解析或等待异步解析完成。
5. 查看切片。
6. 在“智能问答”中提问文档相关问题。

## 资料不足拒答演示

提问：

```text
当前项目是否支持无人机巡检？
```

预期回答：

```text
当前资料不足，无法确认。建议补充相关项目文档、能力清单或历史需求案例。
```
